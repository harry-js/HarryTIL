### Context
사용자께서 'cmux'라는 터미널 도구에 대한 학습 내용을 요청하셨습니다. 'cmux'라는 이름의 유명한 터미널 멀티플렉서는 현재 널리 알려져 있지 않으나, 터미널 멀티플렉서의 필요성과 기능에 대한 관심으로 보아 'tmux'와 같은 도구를 찾으시는 것으로 해석됩니다.

터미널 멀티플렉서는 하나의 터미널 창에서 여러 개의 독립적인 터미널 세션을 관리할 수 있도록 해주는 유틸리티입니다. 이는 특히 원격 서버에 접속하여 작업할 때 네트워크 연결이 끊어지더라도 작업 중인 세션이 유지되도록 하여 생산성을 크게 높여줍니다.

본 문서에서는 널리 사용되는 터미널 멀티플렉서인 `tmux`를 기준으로 설치 방법과 기본적인 사용법을 정리합니다.

### Core

### 설치 방법
*   `tmux`는 대부분의 운영체제에서 패키지 관리자를 통해 쉽게 설치할 수 있습니다.

```bash
# Debian/Ubuntu 기반 시스템
sudo apt update
sudo apt install tmux

# CentOS/RHEL 기반 시스템
sudo yum install tmux  # 또는 sudo dnf install tmux

# macOS (Homebrew 사용)
brew install tmux
```

### 기본 사용법
*   `tmux`의 모든 명령어는 기본적으로 접두사 키(`Prefix Key`)와 함께 사용됩니다. 기본 접두사 키는 `Ctrl+b`입니다.

*   **새로운 세션 시작**
    *   단순히 `tmux` 명령어를 입력하면 새로운 세션이 시작됩니다.
    *   특정 이름으로 세션을 시작하려면:
        ```bash
tmux new -s my_session
        ```

*   **세션 분리(Detach) 및 재연결(Attach)**
    *   현재 작업 중인 세션에서 빠져나와 셸로 돌아가려면:
        *   `Ctrl+b` 누른 후 `d`
    *   분리된 세션에 다시 연결하려면:
        ```bash
tmux attach -t my_session
        ```
    *   마지막으로 사용한 세션에 연결하려면:
        ```bash
tmux attach
        ```
    *   현재 실행 중인 모든 `tmux` 세션 목록을 확인하려면:
        ```bash
tmux ls
        ```

*   **창(Window) 관리**
    *   `Ctrl+b` 누른 후 `c`: 새로운 창 생성
    *   `Ctrl+b` 누른 후 `n`: 다음 창으로 이동
    *   `Ctrl+b` 누른 후 `p`: 이전 창으로 이동
    *   `Ctrl+b` 누른 후 `0`~`9`: 특정 번호의 창으로 이동
    *   `Ctrl+b` 누른 후 `x`: 현재 창 닫기 (확인 메시지 표시)

*   **패인(Pane) 관리**
    *   `Ctrl+b` 누른 후 `%`: 현재 패인을 세로로 분할
    *   `Ctrl+b` 누른 후 `"`: 현재 패인을 가로로 분할
    *   `Ctrl+b` 누른 후 `방향키`: 패인 간 이동
    *   `Ctrl+b` 누른 후 `z`: 현재 패인을 전체 화면으로 토글
    *   `Ctrl+b` 누른 후 `x`: 현재 패인 닫기 (확인 메시지 표시)

### Insight
`tmux`와 같은 터미널 멀티플렉서는 개발자의 생산성을 크게 향상시킬 수 있는 필수 도구입니다. 특히 원격 서버 작업 시 네트워크 연결이 끊어져도 작업 세션이 유지된다는 점은 매우 강력한 이점입니다. 여러 작업을 동시에 진행하거나, 특정 스크립트 실행 후 다른 작업을 해야 할 때 효율적인 작업 환경을 제공하며, 깔끔하게 정리된 터미널 환경을 통해 집중도를 높일 수 있습니다. 초기 학습 곡선이 있을 수 있지만, 그 효용성은 투자할 가치가 충분합니다.
